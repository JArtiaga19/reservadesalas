# Endpoints de la API
(Borrador inicial — se completará en la TAREA 1)

## Endpoint de salud 
GET /health Descripción: Comprueba que la aplicación está levantada y responde 200.

## Endpoints: salas 
GET /salas
Descripción: Obtiene la lista de todas las salas.

GET /salas/{id}
Descripción: Obtiene los datos de una sala específica.

POST /salas
Descripción: Crea una nueva sala.

PUT /salas/{id}
Descripción: Actualiza los datos de una sala existente.

DELETE /salas/{id}
Descripción: Elimina una sala.
